✅ PHASE 2: Inventory Logic (Flexible but Intelligent)
Since double booking is allowed:
We design Soft Conflict Warnings, not hard blocks.

🔹 Availability Calculation Logic
When adding items to an event:
Query all confirmed events for that date


Sum quantities per item


Compare to total_quantity


If requested + existing > total_quantity:


Set overbook_warning = true


Show admin warning:
 ⚠ “Requested 50 chairs. 42 already reserved. Total owned: 60. This will exceed available stock.”


Patrick can proceed anyway.
